# 프로젝트 개요

## 목적
이 프로젝트는 간단한 Idle 전투 흐름을 기반으로 한 미니 게임을 만드는 것을 목표로 합니다.
Phaser 기반 씬에서 전투 상태를 주기적으로 갱신하고, 데이터 중심의 밸런스 설정으로
성장 곡선과 보상을 관리하는 구성을 갖추고 있습니다.

## 구성 요약
- `src/core/combat.js`: 전투 상태 생성과 틱 기반 전투 루프를 담당합니다.
- `src/logic/battleBalance.js`: 밸런스 데이터에 기반해 몬스터 스탯, 보상, 장비 보너스를 계산합니다.
- `src/data/balance.js`: 몬스터/성장 곡선/장비 티어 등 밸런스 데이터의 원천을 제공합니다.
- `src/scenes/GameScene.js`: 전투 상태를 업데이트하고 HUD에 결과를 표시합니다.

## 주요 로직 설명
### 전투 상태 관리
- `createCombatState`에서 기본 장비/성장 곡선을 기준으로 플레이어와 몬스터 상태를 초기화합니다.
- `tickCombat`는 프레임 간 경과 시간에 따라 공격 쿨다운을 줄이고, 공격 타이밍이 도달하면
  피해를 계산해 상태를 갱신합니다.
- 몬스터가 처치되면 `applyKillRewards`를 통해 보상과 처치 수를 반영하고 다음 몬스터를
  생성합니다.

### 밸런스/성장 곡선
- 몬스터 스탯은 `growthCurves` 설정에 따라 선형 혹은 지수 성장으로 계산됩니다.
- 보상 골드는 처치 수에 비례한 보정 배율을 적용해 증가합니다.
- 장비 티어는 공격력/체력 보너스를 추가해 플레이어의 기본 스탯을 확장합니다.

### 화면 표시
- GameScene은 매 프레임 전투 상태를 갱신한 뒤, 플레이어/몬스터 HP, 스테이지, 골드,
  처치 수를 HUD 텍스트로 표시합니다.

## 작업 계획
- 전투 로그(최근 피해/보상)를 저장해 UI에 간단한 히스토리 패널을 추가합니다.
- 장비 업그레이드(티어 선택) UI를 도입해 플레이어 성장 선택지를 제공합니다.
- 몬스터 타입/성장 곡선 선택을 노출해 난이도 설정을 다양화합니다.
- 데이터 밸런스 조정을 위해 간단한 테스트 시나리오(샘플 전투 시뮬레이션)를 추가합니다.
