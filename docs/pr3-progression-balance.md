# PR3 성장 + 경제 밸런스 설계

## 선택한 성장 곡선
- 비용 곡선: **지수형**
  - 공격력 강화 비용: `10 * 1.18^level`
  - 체력 강화 비용: `12 * 1.16^level`
- 강화 효과:
  - 공격력 강화: 1회당 `+8%`
  - 체력 강화: 1회당 `+10%`

지수형 비용을 선택한 이유:
1. 초반에는 빠르게 구매되어 5~10분 내 체감이 쉽습니다.
2. 중반 이후 구매 간격이 길어져 클릭 인플레이션을 완화합니다.
3. 공격/체력 두 축의 효율이 모두 보이기 때문에 한쪽 몰빵 유인을 줄일 수 있습니다.

## Core-Scene 인터페이스 계약
공통 업그레이드 상태 객체:

```js
{
  gold,
  atk,
  hp,
  atkLevel,
  hpLevel,
}
```

Core 함수 시그니처:
- `getUpgradeCost(type, level)`
- `canAfford(gold, cost)`
- `applyUpgrade(state, type)`
- `calcDps(playerStats)`
- `calcSurvivability(playerStats, monsterStats)`

Scene는 위 함수들을 호출만 하며, 비용/증가 수식을 직접 구현하지 않습니다.

## 체감 지표
- DPS: `atk / (attackCooldownMs / 1000)`
- 생존력(예상 생존 시간): `hp / (monsterDps)`

UI에서 현재 스탯, 다음 강화 비용, DPS, 생존 시간을 동시에 노출해
업그레이드 성공 시 즉시 체감할 수 있도록 구성했습니다.
